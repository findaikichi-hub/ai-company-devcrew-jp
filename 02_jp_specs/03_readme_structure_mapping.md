# 03_readme_structure_mapping

## 目的
本家 DevCrew_s の README 構造を参照しながら、日本語版 README にどう写すかを整理する。

## 前提
- 本家 README は参照原本として扱う
- 日本版 README は単純翻訳ではなく、日本企業向けの入口として再設計する
- 日本版 README は、公開・配布・実務説明の起点になることを想定する
- 詳細仕様は README に詰め込みすぎず、各個別ファイルへ分離する

---

## 1. READMEの役割整理

| 観点 | 本家README | 日本版README |
|---|---|---|
| 役割 | リポジトリ全体の入口 | 日本版プロジェクト全体の入口 |
| 読み手 | 本家利用者 / 開発者 / 貢献者 | 日本企業の導入検討者 / 利用者 / 公開閲覧者 |
| 目的 | 構造説明、エージェント一覧、プロトコル一覧、導入導線 | 目的、対象、価値、構造、使い方、差分方針の説明 |
| 詳細度 | 技術仕様への導線が多い | 実務利用の理解を優先し、詳細は別紙へ逃がす |

---

## 2. 本家READMEの要素と日本版への写し方

| 本家の章 / 要素 | 日本版での扱い | 方針 | メモ |
|---|---|---|---|
| リポジトリ概要 | 残す | 日本版の目的説明に置き換える | 日本企業向けローカライズであることを明記 |
| specification-only repository の説明 | 残す | 日本版でも明記する | 実装と仕様を分ける考え方は維持 |
| エージェント一覧 | 残す | 日本版の初期テンプレ一覧に再編する | 20本をそのまま見せるより初期スコープを優先 |
| プロトコル一覧 | 残す | 日本版の参照中核として扱う | 日本独自プロトコル追加予定を明記する |
| ディレクトリ構造説明 | 残す | 日本版リポジトリ構造として再整理する | local / Git / 原本 / 日本版の役割を明確化 |
| セットアップ / 実行手順 | 変える | 初期は最小限にする | まずは仕様整理優先 |
| コントリビューション | 変える | 個人運用前提で簡略化する | 後で公開用に拡張可能 |
| セキュリティ / ガバナンス | 残す | 日本版でも重要項目として残す | 個人情報・監査証跡を補強 |
| 実装リポジトリへの導線 | 変える | 日本版では将来項目にする | 初期は仕様資産の整備を優先 |

---

## 4. 日本版READMEで強めるべき観点

| 観点 | 理由 | 反映方針 |
|---|---|---|
| 日本企業向けであること | 単なる翻訳と誤認されないため | 冒頭で明示する |
| 実務で使う資産であること | 公開用サンプルで終わらせないため | 想定用途を書く |
| 本家準拠であること | 独自暴走を避けるため | 原本参照の方針を明記する |
| 差分管理をすること | 後で変更理由を追えるようにするため | decision_log / changelog を案内する |
| 日本独自の追加要素 | このプロジェクトの価値そのものだから | 敬語、稟議、承認、監査などを明記する |
| 公開は段階的であること | 現状 public でも内容の成熟度は段階的だから | 公開方針を分けて書く |

---

## 5. 日本版READMEでは分離したほうがよい内容

| 内容 | READMEに書くか | 分離先 |
|---|---|---|
| 詳細仕様 | 書きすぎない | 02_jp_specs 以下の個別ファイル |
| 変更理由の詳細 | 概要だけ | 03_decisions/decision_log.md |
| 差分履歴の詳細 | 概要だけ | 04_changelog/CHANGELOG.md |
| 本家の全文説明 | 書かない | 01_original_source の参照で対応 |
| 実装詳細 | 初期は書かない | 将来別資料へ |

---

## 6. 日本版READMEに最初に書くべきメッセージ

- このリポジトリは、DevCrew_s を土台にした日本企業向け AI社員 / AIエージェント仕様テンプレート群である
- 目的は、翻訳ではなく、日本の実務に合う形への再設計である
- 本家原本は参照用として保持し、日本版は差分管理しながら別管理する
- 初期スコープは横断型テンプレートに絞る
- 公開・販売・実務活用につながる資産として育てる
