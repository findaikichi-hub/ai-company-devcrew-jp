# external-collaboration-jp

## 目的
委託先 / 外部連携時の責任境界、共有条件、禁止条件について、日本版の横断統制骨格を定義する。

## 位置づけ
本ファイルは、委託先 / 外部連携管理を扱う governance 文書の skeleton 初稿とする。  
privacy 文書の代替ではなく、外部共有と責任境界に焦点を当てた別文書とする。  
本ファイルで扱う「共有条件」は、外部共有起点の条件 / 責任境界であり、情報種別そのものの機微性判断や取得・利用・保存制約は `privacy-handling-jp.md` 側で扱う。

## 扱う対象
- 委託先連携
- 外部サービス連携
- 社外共有
- 共有条件
- 責任境界
- 禁止条件
- 例外時の扱い

## 基本原則
- 外部共有は必要性と境界を明確にした上で扱う。
- 委託先 / 外部先ごとに責任境界を曖昧にしない。
- 共有可能情報と共有禁止情報を分ける。
- 承認なしの社外共有を許可しない。
- privacy 文書と混ぜず、外部連携の統制として扱う。
- 本ファイルは、誰に出すか、どの範囲で出すか、誰が承認するか、誰が説明責任を負うかという外部共有側の境界を扱う。
- 情報種別そのものの機微性判断や取得・利用・保存制約は `privacy-handling-jp.md` 側で扱う。

## 共有条件の骨格
- 共有目的
- 共有先
- 共有範囲
- 共有期間
- 承認有無
- 記録有無

## 責任境界の骨格
- 誰が共有判断を行うか
- 誰が最終承認を行うか
- 誰が説明責任を負うか
- 誰が停止判断を行うか

## 禁止事項の骨格
- 無承認共有
- 共有先不明の連携
- 共有範囲不明の送信
- 禁止情報の社外送出
- 委託条件未確認の接続

## HITL 接続の骨格
- 本ファイルでいう「HITL 接続」は、HITL条件・HITLトリガー・HITL制御のうち、外部連携統制文脈で必要な人間介入接続を指す。
- HITL は、外部共有の条件確認と接続理由までに留める。
- 状態遷移、承認経路、逐次手順は protocol 側で扱う。
- 社外共有が発生する場合
- 委託先条件が未確認な場合
- 例外連携が必要な場合
- 共有停止判断が必要な場合

## 注記
- 本ファイルは skeleton 初稿であり、委託契約条件や社内外ルール詳細の確定は行わない。
- 個人情報 / 機微情報自体の取扱境界、共有可否、取得・利用・保存制約は `privacy-handling-jp.md` 側で扱う。
- audit evidence や handoff artifact の schema 本文には踏み込まない。
