# privacy-handling-jp

## 目的
個人情報 / 機微情報の扱いについて、日本版の横断統制骨格を定義する。

## 位置づけ
本ファイルは、個人情報 / 機微情報管理を扱う governance 文書の skeleton 初稿とする。  
agent 個別制約ではなく、全体横断の統制境界を定義する。  
本ファイルで扱う「共有時の制約」は、情報種別起点の共有可否 / 制約であり、外部共有時の責任境界そのものは `external-collaboration-jp.md` 側で扱う。

## 扱う対象
- 個人情報
- 機微情報
- 取得時の制約
- 利用時の制約
- 保存時の制約
- 共有時の制約
- 禁止事項

## 基本原則
- 必要最小限の取得・利用・保持を前提にする。
- 機微情報は通常情報と同列に扱わない。
- 目的外利用を許可しない。
- 禁止領域への転記や不要共有を行わない。
- 横断統制として扱い、個別 agent 本文へ分散させすぎない。
- 本ファイルは、何の情報が共有禁止か、何が高機微か、何が人間承認必要かという情報側の境界を扱う。
- 誰に共有するか、どの範囲で共有するか、誰が最終承認責任を負うかは `external-collaboration-jp.md` 側で扱う。

## 境界の骨格
- 取得してよい情報
- 取得前確認が必要な情報
- 保持制限が必要な情報
- 共有禁止情報
- 人間承認が必要な情報

## 禁止事項の骨格
- 不要収集
- 目的外利用
- 無承認共有
- 禁止領域への保存
- 削除・保持条件無視

## HITL 接続の骨格
- 本ファイルでいう「HITL 接続」は、HITL条件・HITLトリガー・HITL制御のうち、個人情報 / 機微情報統制文脈で必要な人間介入接続を指す。
- HITL は、情報種別・利用目的・共有可否の判断条件と接続理由までに留める。
- 状態遷移、承認経路、逐次手順は protocol 側で扱う。
- 高機微情報を扱う場合
- 利用目的が曖昧な場合
- 社外共有が発生する場合
- 例外処理が必要な場合
