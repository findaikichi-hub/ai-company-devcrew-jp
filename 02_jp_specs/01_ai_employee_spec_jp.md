# 01_ai_employee_spec_jp

## 位置づけ
本ファイルは、日本企業向け AI社員 / AIエージェント仕様テンプレートの中核ドラフトとする。

## このテンプレートの目的
- 日本企業で使いやすい AI社員の仕様雛形を定義する
- 本家の設計思想を維持しながら、日本独自要件を追加する
- 各業務別テンプレートの共通土台にする

## 本家から継承する要素
- 役割定義
- 目的 / 責務
- 思考様式
- 記憶の扱い
- 使用可能ツール
- 通信プロトコル
- ガバナンス / 倫理 / 安全
- 運用 / ライフサイクル管理
- ログ / 監査 / 評価 / 改善

## 日本版で追加する要素
- 敬語と対人配慮
- 稟議 / 承認フロー
- 責任分界

- 個人情報 / 機微情報の扱い
- 委託先 / 外部連携管理
- 引き継ぎ前提の記録設計

## 日本版テンプレートの設計原則

### 1. Agent を仕様部品として扱う
- AI社員は単発応答の集合ではなく、仕様に基づく役割部品として定義する
- 各 AI社員は、自分の mandate、responsibility、memory、tools、protocols を持つ

### 2. 反復は protocol を通じて行う
- AI同士の反復は、自由会話ではなく、delegation / handoff / validation / gate を通じて行う
- 初回出力はドラフトとする
- ドラフトは protocol ベースの再委譲・再検証・再提出で改善される

### 3. 完了は protocol 上で決まる
- 生成側AIが自己判断で完了としない
- 完了は quality gate、validation、orchestration によって決まる
- 必要時のみ人間の HITL ゲートを通す

### 4. 構造妥当性は protocol 群で担保する
- 抜け漏れ、重複、粒度不一致、必要パラメータ不足は、validation protocol 群で検出する
- MECE は必要な対象に限定して確認する
- 文章全体に一律適用するのではなく、要件、分解、handoff など構造物を優先対象とする

### 5. 人間は例外責任を持つ
- 人間は常時承認者ではない
- 人間は方針判断、例外承認、HITL ゲート責任者として位置づける

## 日本版で明示すべき必須機能

### 完了判定機能
以下を満たしているかを、gate または orchestration 上で判定する。

- 目的達成条件を満たしている
- 次工程へ渡せる粒度である
- 必須 artifact がそろっている
- 矛盾や重大な欠落がない
- 必要な quality gate を通過している

### 構造検証機能
以下を protocol 群で確認する。

- template validation:
  必須パラメータ、形式、記載漏れ
- decomposition validation:
  タスク分解、責務分解、依存関係の妥当性
- handoff completeness validation:
  handoff 文書、deliverable、次工程条件の充足
- 必要に応じた MECE 確認:
  要件一覧、タスク一覧、責務一覧など
